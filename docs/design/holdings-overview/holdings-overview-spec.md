## 0. 這份東西是什麼

把 planning.md 規劃的「持倉總覽」這條流程，做成**可點擊操作的高保真 prototype**，當作未來 React Native 實作時的**視覺與互動定稿基礎**。涵蓋三個畫面：

1. **持倉總覽（首頁）** — 數據優先的儀表板版型
2. **個別資產詳情** — 點任一持股進入
3. **新增交易** — bottom sheet（右上 ＋ 或詳情頁進入）

> ⚠️ 這是**設計稿/原型**，非正式程式碼。資料為寫死的 mock，沒有 Firebase、沒有真實報價。實作時請依 planning.md 的架構重寫，這裡只定義「長相」與「互動」。

## 1. 探索歷程（為什麼是這個版型）

先用低保真線框並排比較三種「資訊呈現方向」，再收斂：

| 方向           | 結構                                                | 結果    |
| -------------- | --------------------------------------------------- | ------- |
| A 圖表優先     | 大數字 → 大圖表 → 扁平清單（最像 Robinhood）        | 未選    |
| **B 數據優先** | 大數字 → **2×2 摘要卡** → 圖表卡 → 清單（儀表板感） | ✅ 採用 |
| C 組織優先     | 幣別先分開 + 依帳戶分組主導                         | 未選    |

**選 B 的理由**：跨多帳戶、跨幣別的使用情境下，「打開 App 一眼確認今天賺賠 + 整體報酬 + 現金」比「先看一條走勢線」更貼近實際需求。

## 2. 視覺系統（Design Tokens）

### 2.1 色彩

| 角色                 | 值                       | 用途                                 |
| -------------------- | ------------------------ | ------------------------------------ |
| 頁面底               | `#0A0C10`                | 最外層背景                           |
| 畫面底               | `#0E1117`                | 手機畫面背景（頂部帶極淡強調色光暈） |
| 文字 主              | `rgba(255,255,255,0.95)` | 標題、數字                           |
| 文字 次              | `rgba(255,255,255,0.62)` | 一般說明                             |
| 文字 弱              | `rgba(255,255,255,0.42)` | label、單位                          |
| 文字 極弱            | `rgba(255,255,255,0.26)` | 註腳、icon 預設                      |
| 分隔線               | `rgba(255,255,255,0.09)` | row 線、卡片邊                       |
| **強調色（accent）** | `#7C6CF0`（預設）        | 主互動、圖表、active 狀態、光暈      |
| 漲（up）             | `#2FD37E`                | 報酬/今日為正（美式：漲綠）          |
| 跌（down）           | `#FF5E62`                | 報酬/今日為負（美式：跌紅）          |

**強調色可選**（Tweaks 內切換，皆為藍紫線）：`#4C6FE8`（藍）/ `#6C6CF0` / `#7C6CF0`（預設）/ `#A368F0`（紫）。

### 2.2 買入 / 賣出按鈕色（重要決策）

**不採用**傳統台股「買紅賣綠」，改用貼合 App 主調的漸層（取自圓環概念配色）：

- **買入**：`linear-gradient(135deg, #7C5CE6, #C24FD6)`（紫 → 洋紅）
- **賣出**：`linear-gradient(135deg, #2E74E6, #35C6EA)`（藍 → 青）

> 理由：使用者偏好整體統一在藍紫色系，買賣用兩條不同的主題漸層做區分，比紅綠更一致、更現代。
> 注意這與「漲跌數字色」是**兩套獨立系統**：漲跌仍用綠/紅。

### 2.3 字體

- **數字 / 拉丁字**：`Nunito`（500–900）— 圓潤感，對齊原 Gemini 設定的 SF Pro Rounded 調性。所有數字加 `tabular-nums`。
- **中文**：`Noto Sans TC`（400–800）。
- 程式中以 `.num` class 套用數字字體（`--num-font` 變數）。

### 2.4 間距 / 圓角 / 質感

- 間距：8px 網格為主（頁面左右 padding 20px，卡片內 13–16px）。
- 圓角：手機畫面 42 / 卡片 16–18 / bottom sheet 頂 26 / 圓鈕 50% / 膠囊 100。
- **fintech 層次**（克制使用）：
  - 卡片：`linear-gradient(160deg, rgba(255,255,255,.055), rgba(255,255,255,.022))` + 1px 邊 + 柔和外陰影 + 頂部 1px 內高光。
  - 圖表：線條走 accent 漸層 + 底部面積漸層 + 可選 `feGaussianBlur` 光暈（Tweaks 可關）。
  - 強調卡（總報酬率、計算預覽）：帶 accent 色光暈陰影。

## 3. 畫面規格

### 3.1 持倉總覽（首頁 · B 儀表板）

由上到下：

1. **狀態列** 9:41 + 訊號/電量。
2. **Header**：標題「持倉總覽」(23px / 800) + 右側 🔔 通知圓鈕 + ＋ 新增交易（accent 填色圓鈕）。
3. **總資產 Hero**：
   - label「總資產（TWD）」
   - 大數字 `NT$ 1,238,540`（38px / 800，**開啟時 count-up 動畫**，約 0.95s ease-out）
   - 變動行：`▲ NT$ 88,200`　`+7.66%`　`全期`
4. **2×2 摘要 Bento**（gap 10）：
   - 總報酬率 `▲ 7.66%`（強調卡，帶光暈）
   - 總未實現損益 `▲ NT$ 88,200`
   - 今日損益 `▲ NT$ 3,420` / `+0.28%`
   - 本月已實現損益 `▲ NT$ 12,400`
5. **走勢圖卡片**：標題「資產走勢」+ 目前 timeframe；平滑線圖（假資料）；時間 tabs `1M / 3M / YTD / 1Y / ALL`。
6. **Segmented**：`持股 / 帳戶 / 類別`，切換清單分組方式。
7. **持股列表**（三段式 row）：左 = 代號圓標（單字/代號，漸層底）；中 = 名稱 + 代號 / 「N 股 · 均價 X」；右 = 市值 / 漲跌%。
   - 「帳戶」模式：依券商分組（標題顯示檔數 + 幣別小計，**無顏色圓點**）。
   - 「類別」模式：台股 TWD / 美股 USD 分組。
8. **底部 Tab Bar**：持倉 / 交易 / 帳戶 / 設定（4 tab，持倉 active）。其餘 tab 為聚焦用空狀態。

### 3.2 個別資產詳情（push 全屏）

1. Header：返回 ‹ + 中央「代號 名稱 / 英文名 · 台股|美股」+ ⋯ 更多。
2. 現價區：label「目前股價」+ 大數字 + 今日漲跌（金額 + %）+「資料延遲 15 分鐘 · Yahoo Finance」。
3. 走勢圖（較高）+ 時間 tabs `1D / 1W / 1M / 3M / 1Y / ALL`。
4. **TWD / USD 幣別切換**（segmented）：即時換算均價 / 市值 / 未實現損益（demo 匯率 1 USD = 30.95 TWD）。
5. **我的持倉卡**：持有股數 / 平均成本 / 市值 / 未實現損益（金額 + %）/ 帳戶分布（**僅名稱，無色塊**）。
6. 按鈕：「＋ 為此標的新增交易」（帶入該股）/「查看完整交易歷史」。

### 3.3 新增交易（bottom sheet）

1. 頂部 drag handle + Header：取消 / 「新增交易」/ 儲存（accent）。
2. **買入 / 賣出 切換**（主題色漸層，見 §2.2）。
3. 表單：股票代號（可搜尋）、帳戶、交易日期、股數、單價、手續費。
   - 從詳情頁進入會**自動帶入該個股 + 帳戶 + 均價**。
4. **計算預覽卡**（即時）：總成本（買）/ 總收入（賣）= 股數 × 單價 ± 手續費；下方換算 USD。
5. 儲存 → 關閉 + toast「交易已新增（demo）」。

## 4. 互動規格

| 互動                        | 行為                                      |
| --------------------------- | ----------------------------------------- |
| 點持股 row                  | push 進「資產詳情」（右滑入，0.3s）       |
| 返回                        | pop 回首頁                                |
| 右上 ＋                     | 開「新增交易」sheet（空白）               |
| 詳情頁「為此標的新增交易」  | 開 sheet 並帶入該股                       |
| sheet 遮罩 / 取消           | 關閉 sheet                                |
| 時間 tabs                   | 切換走勢圖資料                            |
| Segmented（持股/帳戶/類別） | 重新分組清單                              |
| TWD/USD                     | 即時換算詳情頁數字                        |
| 股數/單價/手續費輸入        | 即時重算總成本                            |
| 開啟首頁                    | 總資產數字 count-up（rAF + timeout 保底） |
| 切換底部 tab                | 其餘為聚焦空狀態（本階段只做持倉流程）    |

## 6. 設計決策記錄

| #   | 決策                                                 | 說明                                            |

| D1  | 採 B（數據優先）版型                                 | 跨帳戶/跨幣別下「一眼看完關鍵數據」優先於走勢線 |

## 7. 與 React Native 實作的對應（建議）

對齊 planning.md §12 的 feature-based 結構：

| 設計元素                                                    | 建議落點                                                                          |
| ----------------------------------------------------------- | --------------------------------------------------------------------------------- |
| 色彩 / 字體 / 間距 tokens                                   | `core/theme/`                                                                     |
| Card / Segmented / TimeTabs / Avatar / 圓鈕 / Field / Input | `core/ui/`                                                                        |
| 走勢圖                                                      | `core/ui/`（MVP 可先用 placeholder，第二階段接真實資料）                          |
| 持倉總覽畫面                                                | `features/holdings/screens/HoldingsOverview`                                      |
| 資產詳情                                                    | `features/holdings/screens/AssetDetail`                                           |
| 新增交易 sheet                                              | Modal group `AddTransaction`（route params 預填 account_id / symbol，呼應 §11.3） |
| count-up / toast 等微互動                                   | `core/ui/` 動畫元件                                                               |

> 注意：本 prototype 的數字格式、漲跌符號（▲▼）、幣別前綴（NT$ / US$）可直接沿用為顯示規則。Money 精度仍須照 planning.md 用 decimal.js，本稿僅為顯示層示意。
